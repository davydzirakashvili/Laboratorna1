# Лабораторна робота №1

## Опис роботи

Дана лабораторна робота присвячена дослідженню можливостей Reflection API та dynamic proxy у мові програмування Java.

У проєкті реалізовано:
- аналіз класів під час виконання програми;
- отримання інформації про поля, методи та конструктори;
- динамічне створення об’єктів;
- виклик методів через Reflection API;
- аналіз стану об’єктів;
- роботу з масивами та зміною їх розмірів;
- dynamic proxy та профілювання методів.

Метою роботи є:
- закріплення навичок програмування на Java;
- ознайомлення з Reflection API;
- використання dynamic proxy;
- організація структури Java-проєкту;
- робота з Git-репозиторієм;
- створення документації до проєкту.

---

# Архітектура проєкту

Проєкт побудований на основі Reflection API та складається з декількох package.

## Структура проєкту

```text
src
│
├── app
│   └── Main.java
│
├── arrays
│   └── ArrayUtils.java
│
├── exceptions
│   └── FunctionNotFoundException.java
│
├── proxy
│   ├── Calculator.java
│   ├── CalculatorImpl.java
│   └── ProxyHandler.java
│
├── reflection
│   └── ReflectionUtils.java
│
└── testclasses
    └── TestClass.java
```

# Опис основних package

## app

Містить головний клас програми `Main.java`, який використовується для запуску та тестування завдань лабораторної роботи.

Усі завдання лабораторної роботи реалізовані в одному проєкті. Для зручності тестування окремих завдань частина коду тимчасово приховується за допомогою коментарів у класі `Main.java`.

---

## reflection

Містить класи для роботи з Reflection API:
- аналіз класів;
- отримання інформації про поля, методи та конструктори;
- виклик методів;
- аналіз стану об’єктів.

---

## arrays

Містить класи для:
- створення масивів;
- зміни розмірів масивів;
- перетворення масивів у рядок.

---

## proxy

Містить реалізацію dynamic proxy та `InvocationHandler` для:
- трасування методів;
- профілювання часу виконання;
- перехоплення викликів методів.

---

## exceptions

Містить власні класи виключень, що використовуються у лабораторній роботі.

---

## testclasses

Містить тестові класи для демонстрації роботи Reflection API.

---

# Логіка роботи програми

Програма демонструє можливості Reflection API та dynamic proxy у Java.

## Основні можливості

1. Аналіз структури класів.
2. Динамічне створення об’єктів.
3. Виклик методів через Reflection API.
4. Аналіз стану об’єктів.
5. Робота з масивами та зміною їх розмірів.
6. Трасування та профілювання методів за допомогою dynamic proxy.

# Примітка

Для перевірки окремих завдань частина коду в `Main.java` може тимчасово коментуватись. Це дозволяє тестувати кожне завдання окремо без впливу попередніх реалізацій.
