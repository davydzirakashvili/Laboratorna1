# Лабораторна робота 1

## Опис роботи
Дана лабораторна робота присвячена розробці Java-застосунку з використанням об’єктно-орієнтованого підходу. У проєкті реалізована базова архітектура застосунку, логіка обробки даних та взаємодія між окремими класами.

Метою роботи є:
- закріплення навичок програмування на Java;
- використання принципів ООП;
- організація структури проєкту;
- робота з Git-репозиторієм;
- створення документації до проєкту.

---

# Архітектура проєкту

Проєкт побудований за модульним принципом та складається з декількох основних компонентів:

## 1. Головний клас (`Main`)
Відповідає за запуск програми та ініціалізацію основної логіки застосунку.

Основні задачі:
- запуск програми;
- створення необхідних об’єктів;
- виклик основних методів;
- обробка взаємодії з користувачем.

---

## 2. Класи моделей (`Model`)
Містять структури даних та описують об’єкти предметної області.

Функції:
- збереження даних;
- інкапсуляція полів;
- надання getter/setter методів;
- представлення сутностей програми.

---

## 3. Сервісні класи (`Service`)
Реалізують основну бізнес-логіку застосунку.

Функції:
- обробка даних;
- виконання обчислень;
- перевірка коректності введених значень;
- взаємодія між різними компонентами системи.

---

## 4. Допоміжні класи (`Utils`)
Містять допоміжні методи для спрощення роботи програми.

Приклади:
- форматування даних;
- робота з файлами;
- допоміжні математичні операції;
- генерація службових даних.

---

# Логіка роботи програми

## Основний алгоритм

1. Користувач запускає програму.
2. Головний клас ініціалізує необхідні компоненти.
3. Вводяться або завантажуються дані.
4. Сервісні класи виконують обробку інформації.
5. Результат повертається користувачу.
6. Програма завершує роботу або переходить до наступної операції.

# Принципи реалізації

У проєкті використані базові принципи об’єктно-орієнтованого програмування:

- Інкапсуляція
- Наслідування
- Поліморфізм
- Абстракція

Також застосовано розділення логіки на окремі класи для покращення читабельності та підтримки коду.
